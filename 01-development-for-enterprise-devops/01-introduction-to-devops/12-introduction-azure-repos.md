# Introduction to Azure Repos

## Overview
Azure Repos is a set of **version control tools** for managing code, suitable for projects of any size.

## Version Control Types

| Type | Model | Description |
|------|-------|-------------|
| **Git** | Distributed | Industry-standard distributed version control |
| **TFVC** | Centralized | Team Foundation Version Control (centralized) |

## Key Features

### Free Private Git Repositories
- Unlimited private Git repository hosting
- TFVC support
- Scales from hobby project to world's largest repository
- Pull requests for code review
- Code search capabilities

### Any Git Client Support
- Securely connect from any IDE, editor, or Git client
- Push code into Git repository
- Cross-platform compatibility

### Extensibility
- **Web hooks**: Add validations and custom integrations
- **REST APIs**: Build custom extensions
- **Marketplace**: Install extensions for additional features

### Code Search & Discovery
- **Semantic code search**: Code-aware search understanding classes and variables
- Quickly find what you're looking for
- Navigate codebase efficiently

### Collaboration Features
- **Threaded discussions**: Effective Git code reviews
- **Continuous Integration**: For each change
- **Forks**: Promote inner source workflows
- **Pull requests**: Collaborative code review process

### Quality & Automation
- **Branch Policies**: Require reviewer sign-out, successful builds, passing tests before merging PRs
- **CI/CD Integration**: Automatically trigger builds, tests, deployments (Azure Pipelines or custom tools)
- **Customizable policies**: Maintain team's high standards

### Tool Compatibility
- Use with favorite editor and IDE
- Support for both Git and TFVC repositories
- Flexible integration options

## Branch Protection Features
- Code reviewer sign-out required
- Successful build required
- Passing tests required
- Customizable to team standards
- Maintains code quality automatically
