# Introduction to GitHub

## Overview
- **World's largest open-source community** (40+ million developers)
- **Microsoft-owned** development platform
- **Git repository hosting** with web-based graphical interface
- **Free for open source** projects

## Core Features by Category

### 1. Automate from Code to Cloud

#### GitHub Actions (Built-in CI/CD)
- Build, test, deploy in same place as code
- Trigger Actions from any GitHub event to any API
- Build in any language or use community workflows
- **Free for open source** with unlimited data transfer

#### GitHub Packages
- Automatically publish package versions
- Install packages/images in CI/CD workflows
- Integrates with preferred package registries
- Free for open source projects

### 2. Security Features

| Feature | Description |
|---------|-------------|
| **Vulnerability Alerts** | Continuous scanning for security advisories; alerts to maintainers with remediation details |
| **Automatic Updates** | Monitors dependencies, opens PRs to update to safe versions |
| **CVE Tracking** | Stay updated with Common Vulnerabilities and Exposures via GitHub Advisory Database |
| **CodeQL** | Industry-leading semantic code analysis; treats code as data to identify vulnerabilities faster |
| **Proactive Scanning** | Prevents vulnerabilities from reaching production |
| **Token Protection** | Detects accidentally committed tokens (20+ service providers supported) |

### 3. Code Review Tools

**Built-in review process**:
- **Pull Requests**: Living conversations about changes
- **Request Reviews**: Get detailed feedback from peers
- **Diffs**: Compare versions highlighting changes
- **Threaded Comments**: Discussions within code
- **Clear Feedback**: Specify required changes vs. suggestions
- **Branch Protection**: Require status checks before merge

### 4. Code & Documentation Hosting

**Repositories**:
- Hundreds of millions of private, public, and open-source repos
- Unlimited private repositories for individuals/groups
- Git LFS support for large files

**GitHub Pages**:
- Host documentation from repositories
- Jekyll static site generator
- Publish from `/docs` folder on main branch

### 5. Project Management

| Tool | Purpose |
|------|---------|
| **Projects** | Task boards living close to code |
| **Issues** | Track bugs, discuss ideas with @mentions, distribute work |
| **Task Boards** | See project's big picture, focus team efforts |

### 6. Team & Community Management

**Team Features**:
- Organize with GitHub teams
- Administrative roles and permissions
- Nested teams for fine-tuned access

**Moderation Tools**:
- Issue and pull request locking
- User blocking for open-source projects
- Focus on productive conversations

**Community Guidelines**:
- Standard codes of conduct (customizable)
- Pre-written licenses
- Set roles and expectations easily
